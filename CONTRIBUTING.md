# Contributing to Claude Memory Bank Plugin

Thank you for considering contributing to the Claude Memory Bank Plugin! We welcome contributions from the community.

## How to Contribute

### Reporting Issues
- Use the GitHub issue tracker to report bugs or suggest features
- For security vulnerabilities, please refer to our [SECURITY.md](SECURITY.md) file
- When reporting issues, please include:
  - Clear description of the problem
  - Steps to reproduce
  - Expected vs actual behavior
  - Version of the plugin you're using
  - Any relevant logs or error messages

### Suggesting Enhancements
- Open an issue describing your enhancement idea
- Explain why this enhancement would be useful
- If possible, provide use cases or examples

### Submitting Changes
1. Fork the repository
2. Create a new branch for your feature or bugfix
3. Make your changes
4. Ensure your code follows our style guidelines
5. Add or update tests as appropriate
6. Commit your changes with clear, descriptive messages
7. Push to your fork and submit a pull request

## Development Setup

### Prerequisites
- Claude Code installed
- Basic understanding of shell scripting and markdown

### Getting Started
1. Clone your fork: `git clone https://github.com/your-username/claude-memory-bank-plugin.git`
2. Navigate to the project directory
3. Install dependencies (if any)
4. Run tests to ensure everything works

### Making Changes
- The main plugin logic is in the `plugins/memory-bank/` directory
- Skills are located in `plugins/memory-bank/skills/`
- Hooks are in `plugins/memory-bank/scripts/`
- Documentation is in the root directory

## Code Style

### Shell Scripts
- Use bash for shell scripts
- Follow standard bash formatting conventions
- Include proper error handling
- Add comments for complex logic

### Markdown Files
- Use clear, concise language
- Follow markdown best practices
- Keep line lengths reasonable (under 100 characters)
- Use proper heading hierarchy

### JSON Files
- Use 2-space indentation
- Keep valid JSON format
- Include necessary comments where allowed

## Pull Request Process

1. Ensure your PR addresses a single concern
2. Update documentation as needed
3. Ensure your code passes any existing tests
4. Write tests for new functionality
5. Keep commit history clean
6. Respond to reviewer comments promptly
7. Maintainers will review and merge when ready

## Community

### Code of Conduct
Please note that this project is released with a Contributor Code of Conduct. By participating in this project you agree to abide by its terms.

### Getting Help
- Check the README.md for usage instructions
- Look through existing issues for similar questions
- Ask for clarification in issue comments if needed

## Recognition

Contributors will be acknowledged in:
